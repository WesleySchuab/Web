Transcrição Mas, reparei um detalhe, um ponto de atenção no nosso código. Vou comentar tudo o que já fizemos no código de "Program.cs" usando o atalho "Ctrl + K + C". Em seguida, criar uma nova conta, a conta3. ContaCorrente conta3 = new ContaCorrente() conta3.saldo = -10;COPIAR CÓDIGO O problema é que estamos conseguindo atribuir valor negativo ao campo de saldo. Isso não é bacana. Por que isso acontece? Voltando à classe ContaCorrente, definimos que o campo saldo da classe é público. Então, ele é visível a partir de outros pontos do código. Isso não é legal. Precisamos resolver a visibilidade desse campo. Isso é uma brecha de segurança no código. Imagine o seguinte: no futuro teremos uma equipe trabalhando no mesmo código. Se tivermos pontos de falhas como esse pode acontecer que outro desenvolvedor consiga inserir valor negativo de saldo e isso quebre a lógica de funcionamento do sistema. Vamos corrigir isso agora. No "ContaCorrente.cs" deixaremos o campo saldo como privado: private double saldo = 100;.
